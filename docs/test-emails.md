# Test Emails & Roles

When adding and testing features, use the following seeded emails to log in via magic links. These are generated by the database seeder (`packages/database/prisma/seeders/`).

## Super Admin (`SUPER_ADMIN`)

Use this account to test platform-wide features like approving new gym registrations or viewing all platform tenants.

- **Email:** `admin@bootcamp-starter.local`
- **Name:** Super Admin

## Gym Managers (`ORG_ADMIN`)

Use these accounts to test gym-specific administrative features (e.g., adding classes, creating plans, managing instructors). They are isolated to their own gym tenant.

- **Email:** `admin@ironpeak.example.com` (Iron Peak Fitness)
- **Email:** `admin@flexzone.example.com` (FlexZone Gym)
- **Email:** `admin@mountainwellness.example.com` (Mountain Wellness)
- **Email:** `admin@cityboxing.example.com` (City Boxing)
- **Email:** `admin@velocitysports.example.com` (Velocity Sports)
- **Email:** `admin@profitraining.example.com` (ProFit Training - Suspended Gym)

## Gym Members (`MEMBER`)

Use these accounts to test the member self-service portal (e.g., booking classes, viewing active subscriptions). They have restricted access scoped to their own account.

- **Email:** `alex.johnson@example.com` (Iron Peak Fitness)
- **Email:** `maria.smith@example.com` (Iron Peak Fitness)
- **Email:** `james.brown@example.com` (FlexZone Gym)

## How to use them for feature testing

- **Tenant Isolation:** When building a feature scoped to a gym, log in as `admin@ironpeak.example.com` and create data. Then log out and log in as `admin@flexzone.example.com` to confirm that the data is isolated and not visible.
- **Member Portal:** When testing booking features in Phase B2 or checkout features, log in as `alex.johnson@example.com` to verify the frontend works properly from a standard member's perspective.
