# @repo/db

Prisma schema, generated client, and migrations for Forward-Mena. Backed by PostgreSQL.

This package is the **single source of truth for the database**. The NestJS API
(`apps/api`) is its only consumer — its `PrismaService` extends the `PrismaClient`
exported here. The web app never touches the DB directly; it talks to the API over HTTP.

## Usage

```typescript
// In apps/api
import { PrismaClient, Prisma } from '@repo/db'; // class + namespace (types)
import { prisma } from '@repo/db'; // ready-to-use singleton (scripts/seeders)
import type { Organization, Subscription } from '@repo/db';
```

## Commands

```bash
npx turbo run db:migrate    # Create + apply a new migration (prompts for name)
npx turbo run db:deploy     # Apply pending migrations only
npx turbo run db:reset      # Drop, recreate, re-migrate, re-seed (destructive)
npx turbo run db:seed       # Run seeders (none defined yet)
npx turbo run db:generate   # Regenerate the Prisma client (@prisma/client under node_modules)
```

The Prisma client is generated by `db:generate` (the standard `prisma-client-js`
location under `node_modules`) and re-exported from `@repo/db`, which `db:build` (`tsc`)
compiles to `dist`. Turbo runs `db:generate` + `db:build` before any app build/dev
(`^db:generate`, `^db:build` in `turbo.json`).

## Schema

[`prisma/schema.prisma`](./prisma/schema.prisma) — single `public` schema. Core models:
`Organization`, `Subscription`, `Payment`, `Event`, `Building`.

## Environment

Migrations need `DATABASE_URL`. Copy `.env.example` to `.env.local` and point it at the
database (local dev uses an SSH tunnel to the shared VPS Postgres — see the repo
infrastructure docs). `db:generate`/`db:build` do not require a live connection.

## Conventions

See [`../../AGENTS.md`](../../AGENTS.md) for the full set. Highlights:

- `cuid()` primary keys, `createdAt`/`updatedAt` on every mutable model.
- `@@index` on every foreign key.
- Never hand-edit migration SQL.
